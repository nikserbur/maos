# MAOS — Monitoring & Optimization System

Платформа «цифрового двойника» производственного предприятия: интерактивная
3D-схема, мониторинг, планирование и оптимизация производства, ИИ-ассистент
на данных предприятия.

Текущий срез — **интерактивная 3D-схема предприятия** на примере линии
глубокой переработки лигнина (брикеты/пеллеты): добыча сырья → грубая очистка →
сушка → тонкая очистка → брикетизация/пеллетизация → склад → сбыт, с котельной
и трансформаторной как вспомогательными узлами.

## Технологический стек

- **React 18 + TypeScript**
- **Vite 5** — дев-сервер и сборка
- **three.js + @react-three/fiber + @react-three/drei** — 3D-визуализация
- **Дизайн-система Palantir Blueprint** (dark-first); токены — `src/styles/tokens.css`

Приложение работает локально, без обращений к внешним сервисам в рантайме.

## Требования

- **Node.js 18+** (рекомендуется 20 LTS)
- **npm 9+**

Проверить версии:

```bash
node -v
npm -v
```

## Установка и запуск

```bash
npm install      # установка зависимостей
npm run dev      # запуск дев-сервера
```

Открыть в браузере: **http://localhost:5173**

### Что должно работать при первом запуске

- Тёмная сцена «цифрового двойника» с синей сеткой-блюпринтом.
- 11 узлов предприятия (3D-модели), соединённых анимированными потоками
  (синий — материальный, оранжевый — энергетический).
- **Наведение** на узел — всплывающая подпись; **клик** — выбор узла и
  правая панель-инспектор с показателями (KPI) и статусом.
- Кольца статусов под узлами: зелёный — в работе, оранжевый — наладка,
  серый — простой, красный — авария.
- Управление камерой: **ЛКМ** — орбита (и выбор узла кликом),
  **колесо** — зум, **ПКМ** — панорама. Клик по пустому месту снимает выбор.

## Команды

| Команда | Назначение |
|---|---|
| `npm run dev` | дев-сервер с HMR (http://localhost:5173) |
| `npm run build` | проверка типов + продакшн-сборка в `dist/` |
| `npm run preview` | локальный предпросмотр собранного `dist/` |
| `npm run typecheck` | только проверка типов TypeScript |

## Структура проекта

```
src/
├── App.tsx                      корневой экран
├── main.tsx                     точка входа
├── styles/
│   ├── tokens.css               дизайн-токены Palantir Blueprint
│   └── global.css               базовые стили
└── features/
    └── plant-scene/             3D-схема предприятия
        ├── PlantScene.tsx       экран сцены (Canvas + UI-оверлеи)
        ├── SceneEnvironment.tsx свет, фон, сетка-пол, OrbitControls
        ├── PlantLayout.tsx      раскладка: узлы + потоки
        ├── StageObject.tsx      узел: модель + статус + подпись
        ├── FlowLinks.tsx        анимированные связи между узлами
        ├── Inspector.tsx        панель показателей выбранного узла
        ├── layout.ts            данные схемы (узлы, координаты, KPI, потоки)
        ├── registry.tsx         тип узла → 3D-компонент
        ├── types.ts             доменные типы
        └── objects/             3D-модели узлов (загрузка .glb)
public/
├── models/                      3D-ассеты предприятия (.glb)
└── textures/                    текстуры
```
