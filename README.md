# MAOS — Monitoring & Optimization System

Платформа мониторинга и оптимизации производственного предприятия:
интерактивная 3D-схема, мониторинг, планирование и оптимизация производства,
ИИ-ассистент на данных предприятия.

Текущий срез — **интерактивная 3D-схема предприятия** на примере
металлургического комбината полного цикла: рудный двор → дробление и
обогащение → аглофабрика → доменная печь → конвертер → прокатный стан →
склад → отгрузка, с ТЭЦ и трансформаторной подстанцией как энергоузлами.
Схема обёрнута в оболочку приложения (рельса навигации, верхняя панель,
статус-бар).

## Технологический стек

- **React 18 + TypeScript**
- **Vite 5** — дев-сервер и сборка
- **three.js + @react-three/fiber + @react-three/drei** — 3D-визуализация
- **Дизайн-система Palantir Blueprint** (dark-first); токены — `src/styles/tokens.css`

Приложение работает локально, без обращений к внешним сервисам в рантайме.

## Архитектура (решение принято)

- **Фронтенд — веб** (React + TypeScript + three.js) — текущий репозиторий.
- **Бэкенд — на C++**: доменное ядро (онтология, Action layer, планировщик/
  оптимизатор), хранение **SQLite + ClickHouse**, ИИ-агент MENTAT.
- Связь — локальный API на `127.0.0.1`; всё локально (local-first).

Обоснование и контур — [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
Сущности, БД, окно входа и прослеживаемость всех документов ТЗ →
[docs/REQUIREMENTS.md](docs/REQUIREMENTS.md). Типы предметной области (контракт
с C++-ядром) — `src/domain/`.

## Требования

- **Node.js 18+** (рекомендуется 20 LTS)
- **npm 9+**

Проверить версии:

```bash
node -v

```

## Установка и запуск

```bash
npm install      # установка зависимостей
npm run dev      # запуск дев-сервера
```

Открыть в браузере: **http://localhost:5173**

### Что должно работать при первом запуске

- Оболочка приложения: слева — рельса навигации (Схема, Мониторинг, План,
  Оптимизация, Сценарии, Администрирование), сверху — панель с переключателем
  темы, снизу — статус-бар. Реализован экран **Схема**, остальные — заглушки.
- Тёмная 3D-сцена предприятия с синей сеткой-блюпринтом.
- 11 узлов предприятия (3D-модели), соединённых анимированными потоками
  трёх типов (синий — материальный, оранжевый — энергетический,
  бирюзовый — газовый).
- **Наведение** на узел — всплывающая подпись; **клик** — выбор узла и
  правая панель-инспектор с показателями (KPI) и статусом.
- Кольца статусов под узлами: зелёный — в работе, оранжевый — наладка,
  серый — простой, красный — авария.
- Управление камерой: **ЛКМ** — орбита (и выбор узла кликом),
  **колесо** — зум, **ПКМ** — панорама. Клик по пустому месту снимает выбор.

## Команды

| Команда | Назначение |
|---|---|
| `npm run dev` | дев-сервер с HMR (http://localhost:5173) |
| `npm run build` | проверка типов + продакшн-сборка в `dist/` |
| `npm run preview` | локальный предпросмотр собранного `dist/` |
| `npm run typecheck` | только проверка типов TypeScript |

## Структура проекта

```
src/
├── App.tsx                      корневой экран → оболочка
├── main.tsx                     точка входа
├── shell/                       оболочка приложения
│   ├── AppShell.tsx             каркас: панель + рельса + контент + статус-бар
│   ├── NavRail.tsx              рельса навигации
│   ├── TopBar.tsx               верхняя панель (+ переключатель темы)
│   ├── StatusBar.tsx            нижний статус-бар
│   ├── screens.tsx              реестр экранов
│   ├── Placeholder.tsx          заглушка нереализованных экранов
│   └── icons.tsx                иконки рельсы
├── styles/
│   ├── tokens.css               дизайн-токены Palantir Blueprint
│   └── global.css               базовые стили
└── features/
    └── plant-scene/             экран «Схема» — 3D-схема предприятия
        ├── PlantScene.tsx       сцена (Canvas + UI-оверлеи)
        ├── SceneEnvironment.tsx свет, фон, сетка-пол, OrbitControls
        ├── PlantLayout.tsx      раскладка: узлы + потоки
        ├── StageObject.tsx      узел: модель + статус + подпись
        ├── FlowLinks.tsx        анимированные связи (материал/энергия/газ)
        ├── Inspector.tsx        панель показателей выбранного узла
        ├── layout.ts            данные схемы (узлы, координаты, KPI, потоки)
        ├── registry.tsx         тип узла → 3D-компонент
        ├── types.ts             доменные типы
        └── objects/             3D-модели узлов (загрузка .glb)
public/
├── models/                      3D-ассеты предприятия (.glb)
└── textures/                    текстуры
```
